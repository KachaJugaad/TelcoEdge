## Claim 1: Weather-Predictive MCS Adjustment Method

A method for predictive modulation and coding scheme (MCS) adjustment in
Canadian rural prairie (RMa) terrain under rain conditions, comprising:

1. Retrieving real-time precipitation intensity data from the Government of
   Canada MSC GeoMet OGC API (anonymous, no API key required, data remains on
   Canadian infrastructure);

2. Evaluating rain intensity against a configurable threshold (default 5 mm/hr)
   and, when the threshold is exceeded, pre-adjusting the MCS index downward by
   a configurable step count (default 2 steps) via an O-RAN E2SM-RC v1.03
   control action, before RF signal degradation occurs at the air interface;

yielding an 11.7% reduction in bit error rate (BER) compared to a fixed-MCS
baseline, as measured by Monte-Carlo simulation over a 3GPP TR 38.901 Rural
Macro (RMa) channel model with ITU-R P.838-3 rain attenuation overlay
(N=50 runs, seed=42, Sionna v0.18.0).

> **Draft note:** The 11.7% BER improvement figure is from a 50-run smoke test.
> A 1000-run full benchmark (human-approved scale-up) is required before this
> figure can be cited externally. All benchmark claims must pass legal review
> per Rule R-9 and H-1.12.

## Claim 2: Canadian Terrain-Specific Channel Model Library

A method for generating terrain-specific RF channel models for Canadian rural
telecommunications coverage under variable weather conditions, comprising:

1. Constructing a library of channel model plugins based on the 3GPP TR 38.901
   Rural Macro (RMa) model, parameterised for four Canadian terrain archetypes:
   prairie, boreal forest, rocky mountain, and arctic tundra;

2. Overlaying ITU-R P.838-3 rain attenuation coefficients onto each terrain
   archetype to produce weather-conditioned channel responses;

yielding a sovereign Canadian RF dataset with full data lineage tracking
(source, date, terrain type, weather condition, operator, Sionna version, and
scene hash recorded per sample), as measured by validation against 3GPP
TR 38.901 Table 7.4.1-1 reference values (Sionna v0.18.0).

> **Draft note:** As of this draft, the prairie_rma archetype is implemented.
> Boreal, mountain, and arctic archetypes are planned for Phases 1-2. Claim
> scope should be reviewed once all four archetypes have benchmark results.

## Claim 3: Sovereign Weather-RAN Integration Pipeline

A method for sovereign weather-informed radio access network control in
Canadian rural terrain under variable weather conditions, comprising:

1. Ingesting real-time weather observations from the Government of Canada
   MSC GeoMet OGC API (operated on Canadian infrastructure, anonymous access,
   no API key required) and routing them through a weather adapter to an O-RAN
   compliant xApp;

2. Translating weather observations into O-RAN E2SM-RC v1.03 control actions
   via a policy engine, where all API calls are logged to a sovereignty
   compliance audit trail (data/api_logs/) and no raw data leaves Canadian
   infrastructure;

yielding a weather-to-RAN control pipeline with full API call logging for
sovereignty compliance, as measured by end-to-end integration testing with
MSC GeoMet live data and O-RAN Near-RT RIC simulation (Sionna v0.18.0).

> **Draft note:** This claim describes the pipeline architecture. Quantitative
> latency and reliability metrics from full integration testing are pending
> (target: Phase 1 demo gate, Week 8). E2E latency budget: Uu <= 3ms,
> RAN <= 2ms, backhaul <= 3ms, application <= 2ms, total <= 10ms.
